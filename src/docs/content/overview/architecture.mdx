## Packages

| Package | Role | Open Source |
|---------|------|-------------|
| `carnot-frontend` | Vite/React SPA — price grid UI, wallet integration, trade placement, docs | No |
| `carnot-backend` | NestJS API — order management, real-time price processing, batch worker, settlement coordination | No |
| `carnot-keeper` | Node.js bot — polls pending batches, invokes SP1 prover, submits proofs to Solana | No |
| `carnot-sdk` | TypeScript library — PDA derivation, IDL types, protocol constants, Pyth feed IDs | **Yes** |
| `carnot-programs` | Anchor/Rust Solana program — vault, deposits, ZK settlement verification, payout distribution | **Yes** |
| `carnot-circuit` | SP1 zkVM Rust circuit — all 28 settlement constraints, Groth16 proving | **Yes** |

## Auth Model

| Caller | Endpoint Group | Auth Method |
|--------|---------------|-------------|
| Frontend | `/api/*` (user-facing) | JWT Bearer (EVM wallet signature challenge) |
| Keeper | `/v1/settlement/*` | `X-API-KEY` header |
| Keeper | `/internal/batch/*` | Bearer token |

Users can't impersonate the keeper, and the keeper can't act on behalf of users. Internal batch endpoints (raw order data) are not exposed on the public API surface.
