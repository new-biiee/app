# SP1 zkVM & Groth16

Carnot's settlement proofs are generated by SP1 (Succinct Labs), a general-purpose ZK virtual machine.

## How SP1 Works

SP1 compiles Rust to **RISC-V** and proves, cryptographically, that a specific execution trace was carried out correctly. The circuit is just a normal Rust program — auditors read Rust, not circuit gadgets.

**Two-phase pipeline:**

1. **STARK phase** — SP1 generates a STARK proof of the RISC-V execution trace. Large (tens of KB), no trusted setup, impractical for on-chain use.
2. **SNARK wrapping** — SP1 wraps the STARK into a **Groth16 SNARK**. Output: a 256-byte constant-size proof and a 32-byte public hash.

The trusted setup for Groth16 is performed once by Succinct Labs for the SP1 wrapping circuit. Carnot inherits this setup — it doesn't run its own ceremony.

## Groth16 Proof Structure

| Field | Type | Size | Description |
|-------|------|------|-------------|
| `proof_a` | G1 point | 64 bytes | $\pi_A$ |
| `proof_b` | G2 point | 128 bytes | $\pi_B$ |
| `proof_c` | G1 point | 64 bytes | $\pi_C$ |

Total: **256 bytes**, constant regardless of batch size or computation complexity.

## Verification Equation

The on-chain verifier checks:

$$e(\pi_A,\, \pi_B) = e(\alpha, \beta) \cdot e\!\left(\sum_i l_i \cdot \gamma_i,\, \gamma\right) \cdot e(\pi_C, \delta)$$

where $e(\cdot,\cdot)$ is the BN254 ate pairing, $\alpha$, $\beta$, $\gamma$, $\delta$ are proving key constants, and $l_i$ are the public inputs — in Carnot's case, just `public_outputs_hash`. If the equation holds, the proof is valid.

## Why Groth16 on Solana

- **Constant size** — 256 bytes always. A proof for 10 trades is the same size as a proof for 10,000.
- **O(1) verification** — Fixed number of BN254 pairing operations regardless of computation size.
- **BN254 precompile** — Solana's runtime has native BN254 support; the pairing check runs in optimized native code, not interpreted Rust.

## Single Public Input: `public_outputs_hash`

Groth16 verification cost scales with the number of public inputs. Carnot collapses all 20 settlement fields into one SHA-256 hash:

```
public_outputs_hash = SHA256(
  batch_id ‖ window_start ‖ window_end ‖ num_trades ‖
  market_regime_id ‖ market_max_multiplier ‖ pyth_checkpoints_hash ‖
  pool_balance_before ‖ pool_balance_after ‖ current_liability_before ‖
  net_payout ‖ keeper_fee ‖ protocol_fee ‖
  num_winners ‖ num_losers ‖ total_winners_payout ‖ total_losers_stake ‖
  payouts_commitment ‖ trades_commitment ‖ nullifier_hash
)
```

The program recomputes this hash from the instruction parameters and checks equality — a proof can't be detached from the data it was generated for.

## Prover Modes

| Mode | Use Case | Speed |
|------|----------|-------|
| `SP1_PROVER=local` | Dev / testing | Slow (tens of minutes on laptop) |
| `SP1_PROVER=network` | Production | Fast (typically seconds to a few minutes per 1-minute batch) |
